## Churn Metrics

### 1. Churn Rate

**Definition**: The percentage of fans who did not renew their membership in a given period.

**Formula**:
```
Churn Rate = (Number of Churned Fans / Total Fans at Start of Period) × 100
```

**SQL Implementation**:
```sql
SELECT 
    ROUND(
        (SUM(CASE WHEN renewal_flag = false THEN 1 ELSE 0 END) * 100.0 / COUNT(*)), 
        2
    ) AS churn_rate_pct
FROM fan_churn_silver
WHERE renewal_flag IS NOT NULL;
```

**Business Context**:
- Benchmark: 40-60% is typical for sports season tickets
- Good: < 30%
- Concerning: > 70%

**Use Cases**:
- Track membership health over time
- Identify high-risk segments
- Measure impact of retention initiatives

---

### 2. Retention Rate

**Definition**: The percentage of fans who renewed their membership.

**Formula**:
```
Retention Rate = (Number of Renewed Fans / Total Fans at Start of Period) × 100

OR

Retention Rate = 100 - Churn Rate
```

**SQL Implementation**:
```sql
SELECT 
    ROUND(
        (SUM(CASE WHEN renewal_flag = true THEN 1 ELSE 0 END) * 100.0 / COUNT(*)), 
        2
    ) AS retention_rate_pct
FROM fan_churn_silver
WHERE renewal_flag IS NOT NULL;
```

---

### 3. Cohort Churn Rate

**Definition**: Churn rate specific to fans who joined in a particular time period.

**Formula**:
```
Cohort Churn Rate = (Churned Fans in Cohort / Total Fans in Cohort) × 100
```

**SQL Implementation**:
```sql
SELECT 
    DATE_TRUNC('month', join_date) AS cohort_month,
    COUNT(*) AS cohort_size,
    SUM(CASE WHEN renewal_flag = false THEN 1 ELSE 0 END) AS churned,
    ROUND(
        (SUM(CASE WHEN renewal_flag = false THEN 1 ELSE 0 END) * 100.0 / COUNT(*)), 
        2
    ) AS cohort_churn_rate
FROM fan_churn_silver
GROUP BY DATE_TRUNC('month', join_date);
```

**Business Insights**:
- Identifies if specific acquisition periods led to lower quality fans
- Helps optimize marketing campaigns by time period
- Reveals seasonality in fan loyalty

## Customer Lifetime Value

### 4. Historical CLV

**Definition**: The total revenue generated by a fan from join date to present.

**Formula**:
```
Historical CLV = Sum of All Revenue from Fan
```

**SQL Implementation**:
```sql
SELECT 
    fan_id,
    total_spend AS historical_clv
FROM fan_churn_silver;
```

**Use Case**: Understanding past value delivered by each fan.

---

### 5. Simple CLV

**Definition**: Basic lifetime value calculation using average tenure and spend.

**Formula**:
```
Simple CLV = Average Annual Revenue × Average Lifespan (Years)
```

**SQL Implementation**:
```sql
WITH metrics AS (
    SELECT 
        AVG(total_spend / NULLIF(years_active, 0)) AS avg_annual_revenue,
        AVG(years_active) AS avg_lifespan_years
    FROM fan_churn_silver
    WHERE years_active > 0 AND total_spend > 0
)
SELECT 
    avg_annual_revenue,
    avg_lifespan_years,
    avg_annual_revenue * avg_lifespan_years AS simple_clv
FROM metrics;
```

---

### 6. Predictive CLV (Discounted)

**Definition**: Forward-looking CLV that accounts for retention probability and time value of money.

**Formula**:
```
Predictive CLV = (Average Annual Revenue × Retention Rate) / (1 + Discount Rate - Retention Rate)

Where:
- Retention Rate = % of fans who renewed (as decimal, e.g., 0.40 for 40%)
- Discount Rate = Cost of capital, typically 10% (0.10)
```

**SQL Implementation**:
```sql
WITH tier_metrics AS (
    SELECT 
        ticket_tier,
        AVG(total_spend / NULLIF(years_active, 0)) AS avg_annual_revenue,
        SUM(CASE WHEN renewal_flag = true THEN 1 ELSE 0 END) * 1.0 / COUNT(*) AS retention_rate
    FROM fan_churn_silver
    WHERE years_active > 0
        AND total_spend > 0
    GROUP BY ticket_tier
)
SELECT 
    ticket_tier,
    avg_annual_revenue,
    retention_rate,
    ROUND(
        avg_annual_revenue * (retention_rate / (1 + 0.10 - retention_rate)),
        2
    ) AS predictive_clv
FROM tier_metrics;
```

**Example Calculation**:
```
Given:
- Average Annual Revenue = $1,200
- Retention Rate = 60% (0.60)
- Discount Rate = 10% (0.10)

CLV = $1,200 × (0.60 / (1 + 0.10 - 0.60))
    = $1,200 × (0.60 / 0.50)
    = $1,200 × 1.20
    = $1,440
```

**Business Interpretation**:
- Higher retention = exponentially higher CLV
- Justifies retention investment
- Helps segment fans by long-term value

---

### 7. Projected N-Year CLV

**Definition**: Expected revenue over next N years for active fans.

**Formula**:
```
Projected N-Year CLV = Historical Spend + (Average Annual Spend × N Years)

Only for fans where renewal_flag = true
```

**SQL Implementation (3-year projection)**:
```sql
SELECT 
    fan_id,
    total_spend AS historical_value,
    CASE 
        WHEN renewal_flag = true THEN
            total_spend + ((total_spend / NULLIF(years_active, 0)) * 3)
        ELSE total_spend
    END AS projected_3yr_clv
FROM fan_churn_silver
WHERE years_active > 0;
```

## Reactivation ROI

### 8. Reactivation ROI

**Definition**: Return on investment for win-back campaigns targeting churned fans.

**Formula**:
```
Reactivation ROI = ((Expected Revenue - Campaign Cost) / Campaign Cost) × 100

Where:
- Expected Revenue = Reactivated Fans × Average Spend After Reactivation
- Campaign Cost = Total Churned Fans × Cost Per Contact
- Reactivated Fans = Total Churned Fans × Reactivation Rate
```

**Detailed Calculation**:
```
1. Campaign Cost = Churned Fans × Cost Per Contact
2. Reactivated Fans = Churned Fans × Reactivation Rate
3. Expected Revenue = Reactivated Fans × Avg Spend After Reactivation
4. Net Profit = Expected Revenue - Campaign Cost
5. ROI % = (Net Profit / Campaign Cost) × 100
```

**SQL Implementation**:
```sql
WITH assumptions AS (
    SELECT 
        'BRONZE' AS tier, 50 AS cost_per_fan, 0.12 AS reactivation_rate, 600 AS avg_spend UNION ALL
    SELECT 'SILVER', 75, 0.15, 800 UNION ALL
    SELECT 'GOLD', 100, 0.18, 1200 UNION ALL
    SELECT 'PLATINUM', 150, 0.20, 2000
),
churned AS (
    SELECT ticket_tier, COUNT(*) AS churned_count
    FROM fan_churn_silver
    WHERE renewal_flag = false
    GROUP BY ticket_tier
)
SELECT 
    c.ticket_tier,
    c.churned_count,
    a.cost_per_fan,
    -- Campaign cost
    c.churned_count * a.cost_per_fan AS total_campaign_cost,
    -- Expected reactivations
    c.churned_count * a.reactivation_rate AS expected_reactivations,
    -- Expected revenue
    (c.churned_count * a.reactivation_rate * a.avg_spend) AS expected_revenue,
    -- ROI
    ROUND(
        (((c.churned_count * a.reactivation_rate * a.avg_spend) - 
          (c.churned_count * a.cost_per_fan)) * 100.0 / 
         (c.churned_count * a.cost_per_fan)),
        2
    ) AS roi_pct
FROM churned c
JOIN assumptions a ON c.ticket_tier = a.tier;
```

**Example Calculation**:
```
Given (Bronze Tier):
- Churned Fans = 1,000
- Cost Per Contact = $50
- Reactivation Rate = 12% (0.12)
- Average Spend After Reactivation = $600

1. Campaign Cost = 1,000 × $50 = $50,000
2. Expected Reactivations = 1,000 × 0.12 = 120 fans
3. Expected Revenue = 120 × $600 = $72,000
4. Net Profit = $72,000 - $50,000 = $22,000
5. ROI = ($22,000 / $50,000) × 100 = 44%
```

**Business Decision Framework**:
- ROI > 100%: Strong candidate for win-back campaign
- ROI 20-100%: Proceed if strategic value exists
- ROI < 20%: Focus on preventing churn instead

## Engagement Metrics

### 9. Engagement Score

**Definition**: Composite metric measuring fan engagement across multiple dimensions.

**Formula**:
```
Engagement Score = (Attendance Score × 40%) + (Recency Score × 30%) + (Spending Score × 30%)

Where:
- Attendance Score = (Games Attended / Max Possible Games) × 100
- Recency Score = Tiered based on days since last login
- Spending Score = Tiered based on total spend
```

**Component Calculations**:

**Attendance Score**:
```
Attendance Score = MIN((Games Attended / 38 Home Games) × 100, 100)
```

**Recency Score**:
```
100 points: Last login ≤ 7 days ago
75 points: Last login ≤ 30 days ago
50 points: Last login ≤ 90 days ago
25 points: Last login ≤ 180 days ago
0 points: Last login > 180 days ago
```

**Spending Score**:
```
100 points: Total spend ≥ $5,000
75 points: Total spend ≥ $2,000
50 points: Total spend ≥ $1,000
25 points: Total spend ≥ $500
10 points: Total spend < $500
```

**SQL Implementation**:
```sql
SELECT 
    fan_id,
    -- Attendance score
    CASE 
        WHEN games_attended IS NULL THEN 0
        ELSE LEAST((games_attended * 100.0 / 38), 100)
    END AS attendance_score,
    -- Recency score
    CASE 
        WHEN days_since_last_login <= 7 THEN 100
        WHEN days_since_last_login <= 30 THEN 75
        WHEN days_since_last_login <= 90 THEN 50
        WHEN days_since_last_login <= 180 THEN 25
        ELSE 0
    END AS recency_score,
    -- Spending score
    CASE 
        WHEN total_spend >= 5000 THEN 100
        WHEN total_spend >= 2000 THEN 75
        WHEN total_spend >= 1000 THEN 50
        WHEN total_spend >= 500 THEN 25
        ELSE 10
    END AS spending_score,
    -- Overall engagement score
    ROUND(
        (
            CASE WHEN games_attended IS NULL THEN 0 ELSE LEAST((games_attended * 100.0 / 38), 100) END * 0.4 +
            CASE 
                WHEN days_since_last_login <= 7 THEN 100
                WHEN days_since_last_login <= 30 THEN 75
                WHEN days_since_last_login <= 90 THEN 50
                WHEN days_since_last_login <= 180 THEN 25
                ELSE 0
            END * 0.3 +
            CASE 
                WHEN total_spend >= 5000 THEN 100
                WHEN total_spend >= 2000 THEN 75
                WHEN total_spend >= 1000 THEN 50
                WHEN total_spend >= 500 THEN 25
                ELSE 10
            END * 0.3
        ),
        0
    ) AS engagement_score
FROM fan_churn_silver;
```

**Engagement Categories**:
- **Highly Engaged**: Score ≥ 70
- **Moderately Engaged**: 40 ≤ Score < 70
- **At Risk**: Score < 40

## Revenue Metrics

### 10. Average Revenue Per User (ARPU)

**Formula**:
```
ARPU = Total Revenue / Total Fans
```

**SQL**:
```sql
SELECT ROUND(AVG(total_spend), 2) AS arpu
FROM fan_churn_silver;
```

---

### 11. Revenue at Risk

**Formula**:
```
Revenue at Risk = Sum of total_spend WHERE renewal_flag = false AND total_spend > threshold
```

**SQL**:
```sql
SELECT 
    SUM(total_spend) AS revenue_at_risk
FROM fan_churn_silver
WHERE renewal_flag = false
    AND total_spend > 2000;  -- High-value fans only
```

---

### 12. Projected Annual Revenue Loss

**Formula**:
```
Annual Revenue Loss = Sum((Total Spend / Years Active)) WHERE churned
```

**SQL**:
```sql
SELECT 
    SUM(total_spend / NULLIF(years_active, 0)) AS projected_annual_loss
FROM fan_churn_silver
WHERE renewal_flag = false
    AND years_active > 0;
```

## Key Assumptions & Benchmarks

### Discount Rate
- **Standard**: 10% (0.10)
- **Conservative**: 15% (0.15)
- **Aggressive**: 5% (0.05)

### Reactivation Rates (by Tier)
- **Bronze**: 10-15%
- **Silver**: 12-18%
- **Gold**: 15-22%
- **Platinum**: 18-25%

### Campaign Costs (per contact)
- **Email**: $5-15
- **Direct Mail**: $30-75
- **Phone Outreach**: $50-150
- **Premium Gift**: $100-300

### Industry Benchmarks
- **Sports Churn Rate**: 30-60%
- **Avg CLV (Season Ticket)**: $5,000-15,000
- **Engagement Score**: 60+ is strong
